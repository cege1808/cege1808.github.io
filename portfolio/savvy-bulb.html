Savvy Bulb 

automated lighting fixture · Jan-Apr'16

The projectâ€™s aim is to build an easy to use light bulb adapter which turns any standard light bulb into a smart light. This allows any light bulb to be remotely controlled.

Components
  • Hardware: AC/DC dimming circuit
  • Firmware: Particle IOT board (old name: Spark)
  • User Interfaces: iOS, Android, Web
  • Enclosure: 3D-printed body

Built the website interface which communicates with the Particle board that controls the dimming circuit.

Final Design

· · · jQuery

The web application is intended for users who own large screen devices, unlike the mobile applications. The app is built using HTML5, CSS and JavaScript. The design must accommodate the major functions (turning on/off and dimming) and additional functions (strobe, alarm and timer). These functions are described in more detailed here. The interface is displayed with cells which are expandable and sortable.



One Cell Display

To create a clean display on a larger screen, each registered light is displayed in a cell. The immediate display of the cells are the name of the light bulb, the on/off switch, and the dimmer slider allows user to quickly change the luminosity of the bulb.


Expandable

On the top right corner of the cell is a button which expands the cell and displays the strobe, timer and alarm features. The expandable feature is used to hide the less commonly used features and highlight the major features.



Sortable

The users are able to arrange the order of the list cells by dragging the cells to the intended position. This allows users to customize the order of their lightbulbs which is independent of the order it was set up.

Future Improvements
 Lessons Learned
  • Focus more on the most important features first. The most important features are communication with the Particle board and the dimming interface, and not the timer, alarm or strobe features
  • Better ways to change the state of a button aside from the toggle function in jQuery
  • Eventhough websites are viewed on larger screens, the slider must be at a comfortable length for users to easily slide
  • Refactor brute force functions into more reusable components
  • Do user interaction interviews to recieve feedback on each components' usability
 Building a Back-End
  • User login that automatically connects to the Particle eco-system
  • Store all the lights in the database by creating models
  • Create Reusable components
 Upgrading the Front-End
  • Replace jQuery with React for faster response times
  • Replace Ajax calls with fetch (and promises)
  • Store light states in React Components and call fetch to update display
  • Improve design by starting with a mobile-first approach
  • Update page by getting the current brightness from the Particle board every # of seconds to be compatible with multiple devices (iOS or Android)
